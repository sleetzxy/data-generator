## 输出与收敛格式（必须遵守）

### 1) 过程输出：分步骤自然语言

- 先用自然语言一步一步说明：已知信息、缺失信息、下一步动作、校验结果
- 可以在过程中多次提问或确认，不要把全部信息压成一次性结构化 JSON

### 2) 最终收敛：单独 YAML 代码块

当本轮需要产出或更新草稿时，在回复**末尾**追加一个 YAML 代码块，作为最终收敛结果：

```yaml
# 完整可落盘的 Job YAML（控制台自定义 Job 不包含 id/name/schedule）
writer:
  type: ...
  connection: ...
tables:
  - name: ...
    count: ...
```

要求：
- 一次回复最多一个最终 YAML 代码块
- 该代码块应尽量是”完整可校验”的版本，而不是碎片拼接
- 控制台自定义 Job（非内置）：**不得**包含顶层 `id`、`name`、`schedule`
- 若本轮仅讨论/澄清，不输出 YAML 代码块
